Orchard FEM conda environment commands
======================================

Principle
---------
1. Keep `orchard_fem` development centered on the active Orchard FEM package.
2. Keep the PETSc/SLEPc/FEniCSx stack in its own environment.
3. Keep PyTorch surrogate / inverse work separable from the solver stack to reduce dependency conflicts.


Environment A: general Python development / tests / plotting
-----------------------------------------------------------
Purpose:
- Run the primary `orchard_fem` workflow
- Run Python integration tests
- Generate plots and benchmark summaries

Command:
conda create -n orchard-dev -c conda-forge python=3.11 pytest pytest-cov ruff mypy numpy scipy pandas matplotlib pyyaml h5py jupyterlab

Activation:
conda activate orchard-dev

Optional:
- If you want PyTorch in this environment too:
  conda install -n orchard-dev -c pytorch -c conda-forge pytorch


Environment B: FEniCSx / PETSc / SLEPc backend
----------------------------------------------
Important:
- Recommended target platform: Ubuntu, WSL2 Ubuntu, or native Linux
- Treat this as the main high-fidelity solver / verification environment

Purpose:
- orchard_fem modal / frequency / time solver
- PETSc / SLEPc eigen and linear solve backend
- future DOLFINx integration

Command:
conda env create -f config/fenicsx_pinn_environment.yml

Activation:
conda activate orchard-fenicsx

Optional:
- If you also want PyTorch here and the solver resolves cleanly:
  conda install -n orchard-fenicsx -c pytorch -c conda-forge pytorch


Environment C: PyTorch inverse / surrogate
------------------------------------------
Purpose:
- orchard_pinn inverse MVP
- local nonlinear surrogate
- response surrogate and dataset work

Command:
conda create -n orchard-ml -c pytorch -c conda-forge python=3.11 pytorch numpy scipy pandas matplotlib h5py pyyaml pytest pytest-cov jupyterlab

Activation:
conda activate orchard-ml


Minimal recommended rollout order
---------------------------------
1. Create orchard-dev first
2. Create orchard-fenicsx inside Ubuntu / WSL2 Ubuntu
3. Create orchard-ml when you start PINN / surrogate work


Post-install checks
-------------------
After activating any environment, run:
python -m orchard_fem doctor
python -m orchard_fem --help
